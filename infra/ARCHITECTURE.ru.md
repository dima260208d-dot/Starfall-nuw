# Архитектура Starfall — бесплатные облака

```
┌─────────────────────────────────────────────────────────────────┐
│  БРАУЗЕР                                                         │
└───┬──────────────┬────────────────────┬─────────────────────────┘
    │              │                    │
    ▼              ▼                    ▼
┌─────────┐  ┌──────────────┐   ┌──────────────┐
│Supabase │  │ Cloudflare   │   │ Диспетчер    │
│         │  │ R2 (+Worker) │   │ (server/)    │
└─────────┘  └──────────────┘   └──────────────┘
 аккаунты      3D, текстуры       команды/пати
 прогресс      (без сервера!)      Render/Oracle
 карты         + Render/Oracle     (отдельно)
               только для боёв WS
```

---

## Разделение (всё бесплатно)

| Данные | Сервис | Настройка |
|--------|--------|-----------|
| Логин, прогресс, карты | **Supabase** | ✅ готово |
| **3D, GLB, тайлы** | **Cloudflare R2** | `VITE_ASSET_CDN_URL` |
| **Бои (WebSocket)** | **Render** или **Oracle** | `VITE_EDGE_SERVER_URL` |
| **Команды / пати** | отдельный хост | `VITE_GAME_SERVER_URL` |

### ❌ Не используем
- **Koyeb** — у многих просит оплату
- **Fly.io** — платный для новых аккаунтов
- **Supabase** — для команд и тяжёлых файлов

---

## Графика — без сервера

3D-модели **не нужно** класть на Koyeb/Oracle.  
R2 отдаёт файлы напрямую с глобальным CDN Cloudflare — это мощнее и бесплатнее.

→ `infra/deploy-graphics.ru.md`

---

## `.env.local` (пример)

```env
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...

# Графика — R2 напрямую (главное!)
VITE_ASSET_CDN_URL=https://pub-xxxx.r2.dev/

# Бои — Render (бесплатно) или Oracle (мощнее)
VITE_EDGE_SERVER_URL=https://starfall-edge.onrender.com

# Команды — позже, отдельный сервер
# VITE_GAME_SERVER_URL=...
```
